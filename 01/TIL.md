# 01

## 사용한 도구 / 명령어

```bash
# 어셈블리 소스를 순수 바이너리(flat binary)로 컴파일
nasm -f bin boot.asm -o boot.bin

# 바이너리를 디스어셈블해서 실제 기계어 확인
ndisasm boot.bin

# QEMU로 가상 디스크로 붙여서 실행
qemu-system-x86_64.exe -hda boot.bin
```

## 부트 섹터(Boot Sector)

컴퓨터가 켜지면 BIOS가 가장 먼저 저장 장치(HDD, USB 등)의 **첫 번째 512바이트**를 읽는다.  
이 512바이트의 마지막 2바이트가 `0x55 0xAA`이면 BIOS는 이걸 유효한 **부트 섹터**로 인식하고,  
해당 512바이트를 메모리 주소 **0x7C00**에 로드한 뒤 실행을 넘긴다.

> 정리: 부트 섹터 = 디스크의 첫 512바이트, BIOS가 메모리에 올려주는 최초 코드

## boot.asm 코드 분석

```nasm
ORG 0x7C00
BITS 16
```

| 지시어       | 의미                                                                                      |
| ------------ | ----------------------------------------------------------------------------------------- |
| `ORG 0x7C00` | 이 코드가 메모리 0x7C00에 로드된다고 NASM에 알림. 레이블/주소 계산의 기준점이 됨.         |
| `BITS 16`    | 16비트 리얼 모드(Real Mode) 코드로 어셈블하라는 지시. BIOS가 넘겨줄 때 CPU는 16비트 모드. |

### BIOS INT 10h - 화면에 문자 출력

```nasm
start:
    mov ah, 0Eh   ; 기능 번호: Teletype Output
    mov al, 'A'   ; 출력할 문자 (ASCII 0x41)
    mov bx, 0     ; BH=페이지 번호(0), BL=색상(그래픽 모드 전용)
    int 0x10      ; BIOS 비디오 인터럽트 호출
```

`INT 10h / AH=0Eh` 는 BIOS가 제공하는 **텔레타입 출력** 루틴이다.  
레지스터에 값을 세팅한 뒤 소프트웨어 인터럽트(`int 0x10`)를 발생시키면  
BIOS가 알아서 화면에 문자를 찍어준다.

참고: https://www.ctyme.com/intr/rb-0106.htm

### 무한 루프

```nasm
jmp $
```

`$`는 NASM에서 **현재 명령어의 주소**를 의미한다.  
즉, `jmp $`는 자기 자신으로 점프 → 무한 루프.  
부트로더가 할 일을 마친 뒤 CPU가 엉뚱한 곳으로 튀어가지 않도록 멈춰두는 관례다.

디스어셈블 결과에서도 확인할 수 있다:

```
00000009  EBFE    jmp 0x9   ; 0x9로 점프 = 자기 자신
```

### 512바이트 패딩 + 부트 시그니처

```nasm
times 510-($ - $$) db 0
dw 0xAA55
```

| 구문             | 의미                                              |
| ---------------- | ------------------------------------------------- |
| `$$`             | 현재 섹션(코드 블록)의 시작 주소                  |
| `$ - $$`         | 지금까지 생성된 코드의 바이트 수                  |
| `510 - ($ - $$)` | 코드 뒤부터 510바이트까지 남은 공간               |
| `times ... db 0` | 그 공간을 0x00으로 채움                           |
| `dw 0xAA55`      | 511~512번째 바이트에 부트 시그니처 기록 (2바이트) |

결과적으로 전체가 정확히 **512바이트**가 된다.

## Little Endian과 부트 시그니처

BIOS는 디스크의 511번째 바이트가 `0x55`, 512번째 바이트가 `0xAA`인지 확인한다.  
Intel CPU는 **Little Endian** 방식으로 메모리에 저장한다 — 낮은 바이트가 낮은 주소에 먼저 온다.

따라서 NASM에서 `dw 0xAA55`라고 쓰면:

- 메모리 511번째(낮은 주소): `0x55`
- 메모리 512번째(높은 주소): `0xAA`

BIOS가 기대하는 순서(`55 AA`)와 정확히 일치한다.

디스어셈블 결과 마지막 줄:

```
000001FD  0055AA    ...
```

오프셋 `0x1FE`(= 510) 위치에 `55 AA`가 있는 것을 확인할 수 있다.

## 전체 메모리 맵 요약

```
0x7C00  ┌───────────────────────────┐
        │  실제 코드 (몇 바이트)      │  ← ORG 0x7C00 기준
        ├───────────────────────────┤
        │  0x00 패딩                │  ← times 지시어로 채움
        ├───────────────────────────┤
0x7DFE  │  0x55  0xAA  (부트 시그니처) │  ← dw 0xAA55 (Little Endian)
        └───────────────────────────┘
         총 512바이트
```

## 핵심 정리

1. BIOS는 디스크 첫 512바이트를 **0x7C00**에 올리고 실행한다.
2. 마지막 2바이트가 `55 AA`여야 BIOS가 유효한 부트 섹터로 인정한다.
3. `ORG 0x7C00`으로 주소 계산 기준을 맞춰줘야 레이블이 올바른 주소를 가리킨다.
4. `BITS 16`으로 리얼 모드 16비트 코드임을 명시한다.
5. `INT 10h / AH=0Eh`로 BIOS의 도움을 받아 화면에 문자를 출력할 수 있다.
6. `jmp $`로 무한 루프를 만들어 부트로더 종료 후 CPU가 방황하지 않게 한다.
