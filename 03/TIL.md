## x86 리얼 모드의 세그먼트:오프셋 주소 체계

x86 리얼 모드에서 CPU는 **20비트 물리 주소**를 아래 공식으로 계산한다:

```
물리 주소 = 세그먼트 레지스터 × 16 + 오프셋
```

예시:

```
CS = 0x07C0, IP = 0x0000  →  0x07C0 × 16 + 0x0000 = 0x7C00  ✓
CS = 0x0000, IP = 0x7C00  →  0x0000 × 16 + 0x7C00 = 0x7C00  ✓
```

두 경우 모두 물리 주소는 같다.  
문제는 **BIOS마다 부트로더를 넘길 때 CS:IP를 다르게 설정**할 수 있다는 것.  
따라서 직접 CS를 고정해줘야 안전하다.

## Far Jump — CS 레지스터 강제 설정

```nasm
jmp 0x7c0:start
```

일반 `jmp`(Near Jump)는 IP만 바꾼다.  
**Far Jump**는 `CS:IP`를 동시에 바꾼다.

- `CS` ← `0x7C0`
- `IP` ← `start` 레이블의 오프셋 (ORG 0 기준이므로 작은 값)

이 명령어 한 줄로 CS가 `0x07C0`으로 확정된다.

## ORG 0 vs ORG 0x7C00

```nasm
ORG 0        ; 03에서 사용
ORG 0x7C00   ; 02에서 사용
```

`ORG`는 어셈블러에게 "이 코드가 메모리 어느 주소에 올라간다고 가정하고 레이블 주소를 계산해라"라고 알리는 것이다.

- `ORG 0x7C00` + `DS=0x0000` → `물리 주소 = 0 + 레이블오프셋(0x7C0x)` = `0x7C0x` ✓
- `ORG 0` + `DS=0x07C0` → `물리 주소 = 0x7C00 + 레이블오프셋(작은 값)` = `0x7C0x` ✓

결국 물리 주소는 동일하다. 03은 DS를 세그먼트 방식으로 운용하는 방식을 택했다.

## 세그먼트 레지스터 초기화

```nasm
cli             ; 인터럽트 비활성화 (초기화 도중 방해받지 않기 위해)
mov ax, 0x7c0
mov ds, ax      ; 데이터 세그먼트: lodsb 등 데이터 접근 기준
mov es, ax      ; 엑스트라 세그먼트: 문자열 복사 등에 사용
mov ax, 0x00
mov ss, ax      ; 스택 세그먼트
mov sp, 0x7c00  ; 스택 포인터: 0x0000:0x7C00 = 물리 주소 0x7C00
sti             ; 인터럽트 다시 활성화
```

### 각 레지스터 역할

| 레지스터 | 역할                                                           |
| -------- | -------------------------------------------------------------- |
| `DS`     | 데이터 세그먼트. `lodsb`(`DS:SI`), 일반 메모리 읽기의 기준     |
| `ES`     | 엑스트라 세그먼트. 문자열 복사 명령(`movs` 등)의 목적지 기준   |
| `SS`     | 스택 세그먼트. `push`/`pop`/`call`/`ret`의 기준                |
| `SP`     | 스택 포인터. 스택의 현재 위치. `push`하면 감소, `pop`하면 증가 |

### 스택이 0x7C00인 이유

스택은 **높은 주소에서 낮은 주소 방향으로 자란다.**  
`SS=0x0000`, `SP=0x7C00`이면 물리 주소 `0x7C00`부터 아래로 커진다.  
부트로더 코드는 `0x7C00`에서 위로 있으므로, 서로 반대 방향 → 충돌하지 않는다.

```
0x7C00  ←── 스택 시작 (SP), 아래로 자람
        ...
0x7C0x  ──→ 부트로더 코드, 위로 자람
```

## cli / sti — 인터럽트 제어

```nasm
cli   ; Clear Interrupt Flag — 인터럽트 비활성화
...
sti   ; Set Interrupt Flag   — 인터럽트 활성화
```

세그먼트 레지스터를 초기화하는 도중에 인터럽트가 발생하면  
SS는 바뀌었는데 SP는 아직 안 바뀐 상태 같은 **불일치 상황**이 생길 수 있다.  
`cli`로 인터럽트를 잠시 막아서 초기화 시퀀스가 원자적으로 실행되게 한다.

## 핵심 정리

1. 리얼 모드 물리 주소 = `세그먼트 × 16 + 오프셋`
2. Far Jump(`jmp seg:offset`)로 CS를 원하는 값으로 강제 설정할 수 있다.
3. `ORG 0` + `DS=0x7C0`은 `ORG 0x7C00` + `DS=0`과 동일한 물리 주소를 가리킨다.
4. DS, ES, SS, SP는 BIOS가 보장해주지 않으므로 부트로더가 직접 초기화해야 한다.
5. 스택은 높은 주소에서 낮은 주소로 자란다 — `SP=0x7C00`으로 코드와 충돌을 피한다.
6. `cli`/`sti`로 레지스터 초기화 도중 인터럽트를 차단해 안전하게 설정한다.
